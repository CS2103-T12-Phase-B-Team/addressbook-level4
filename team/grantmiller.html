Undo previous command : undo

Restores the address book to the state before the previous undoable command was executed.
Format: undo

Undoable commands: those commands that modify the address book’s content (add, delete, edit and clear).

Examples:

  • delete 1
    list
    undo (reverses the delete 1 command)

  • select 1
    list
    undo
    The undo command fails as there are no undoable commands executed previously.

  • delete 1
    clear
    undo (reverses the clear command)
    undo (reverses the delete 1 command)

Redo the previously undone command : redo

Reverses the most recent undo command.
Format: redo

Examples:

  • delete 1
    undo (reverses the delete 1 command)
    redo (reapplies the delete 1 command)

  • delete 1
    redo
    The redo command fails as there are no undo commands executed previously.

  • delete 1
    clear
    undo (reverses the clear command)
    undo (reverses the delete 1 command)
    redo (reapplies the delete 1 command)
    redo (reapplies the clear command)

End of Extract

Justification

Autocomplete is a common feature among many applications. It greatly increases the ease of use for a user by speeding up their ability to complete common commands.

My implementation use a dropdown box to visually show the user what possible commands there are for an input, if any, as well as easily enabling the user to see the parameters for their command directly in the command line.

Autocomplete

The autocomplete functionality is accomplished by using a character trie to store all possible commands. The trie has a very simple set of commands, shown below.

public interface Trie {
    String attemptAutoComplete (String input);
    void insert (String input);
    Set<String> getCommandSet();
}

The trie is made up of node classes which only have three fields. The next field represents a character on the same level as the current one. The child represents a next possible letter. For example in the words edit and exit the node for 'd' would have a child 'i' and a next 'x'. 'x' would have a child 'i'.

public class Node {

    private Node next;
    private Node child;
    private char key;
    ...
}

Upon the pressing of the tab key, the command box calls attemptAutoComplete, in the commandTrie class. This method will traverse the trie using the content of the command box. If the input has a character not present in the trie, then the attempt fails. If the trie parses through the input but reaches a node with a next option, it also fails. If the trie parses the input and reaches a node with only children, it will autocomplete the input to a valid command. If the autocomplete fails, the text is turned red.

An added functionality is that if the user presses tab again, the command will add a usage framework for the command, i.e. find will be autocompleted to find [names]. This helps the user to understand command usage. If tab is pressed again, the input simply reverts back to the command.

One class, command collection, is used to hold the set of commands and a map from each command word to the command usage. This is used by the trie to both construct the command trie and autocomplete from a command word to a command word and its usage. This static class simply contains a set of each command’s command word that is statically initialized and used by the commandTrie class.

Start of Extract [from: User Guide]

Person: group

Each person can now be assigned a group. The group is used to sort people into specific tours for the tourguide. Since v1.2

Addressbook Groups: group

The address book stores all user created groups and allows the user to add people into existing groups. Names must be one word and be unique. Since v1.3

Format:

  • group [name] creates a group called [name] if one doesn’t already exist. If one does, it will be deleted.

  • group [name] [names…​] creates a group called [name] and will add the people in the following arguments to it.

Example:

  • group trip Alice Bernice Grant → creates a group called trip with Alice, Bernice, and Grant

  • group trip → deletes the previously created 'trip' group

Group Filter: filter

Ability to filter the UI display to only display selected groups. Bar across the top of the UI displays the current groups. The default filter is none and is applied to everyone in the addressbook. Since v1.3

Format: filter [groupname]

Example:

  • filter none

  • filter trip

End of Extract

Justification

Groups were added to the addressbook so that the owner could easily filter their addressbook between groups of people.
