Enhancement Added: Expiry Date

External behavior

Start of Extract [from: User Guide]

Expiry Date: expire

Set the expiry date of a person. Format: expire INDEX [d/EXPIRY DATE]

Example:
expire 1 d/2017-10-31 Will set the expiry date of the first person to 2017-10-31.
expire 1 Will remove the expiry date of the first person.

By omitting the expiry date field in the command, the expiry date set will be removed.
Expiry date should be in the format YYYY/MM/DD

End of Extract

Justification

Since our product targets tour guides who need to deal with a lot of one-off contact information, expiry date can be used to keep track of when the person will become irrelevant, avoiding problems like duplicate names, or too many persons slowing down the application. Expiry date can also be utilized to implement auto-delete feature. By checking if the expiry date has passed, the application can remove expired contents for the user at startup. ---

Implementation

Start of Extract [from: Developer Guide]

Expiry date

Expiry date indicates the date that information about a person is no longer useful. It is one of the fields of a person alongside name, email, etc. An ExpiryDate class contains a Date Value.

Date validation is implemented in two steps:

  1. Validating the date format with regular expression
    Where the date format is "YYYY-MM-DD"

  2. Validating the date value with SimpleDateFormat object
    DATE_FORMATTER, a SimpleDateFormat object, is created and used for:

Parsing String (from user input) to Date

ParsePosition parsePos = new ParsePosition(0);
this.value = DATE_FORMATTER.parse(date, parsePos);

Formatting Date to String (for printing)

public String toString() {
    return DATE_FORMATTER.format(value);
}

Design Considerations

Aspect: Implementation of ExpiryDate
Alternative 1 (current choice): Store value as Date object
Pros: Methods in Date class can be utilized in implementation of other functions involving ExpiryDate. Date formats doesn’t matter in most methods (eg: comparisons).
Cons: Need to put more work into parsing (from String to Date) and formatting (Date to String).
Alternative 2: Store value as String
Pros: Easy to implement. Easy to print.
Cons: Validation of date as String is complicated. Need to implement all ExpiryDate methods from scratch.

End of Extract

Enhancement Proposed: Add feature autodelete

As mentioned, it is very likely for tour guides to have a lot of persons that are no longer relevant in their addressbook, autodelete can remove expired persons from their addressbook, saving the time needed to delete those persons one by one with delete.
